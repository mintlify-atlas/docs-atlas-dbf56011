---
title: "POST /api/soporte"
description: "Envía un ticket de soporte técnico. También documenta el endpoint compañero GET /api/admin-soporte para consultar los tickets desde el panel de administración."
---

Registra un nuevo ticket de soporte técnico. Este endpoint es invocado por el formulario de soporte del sitio y acepta cuatro categorías de solicitud: `consulta`, `error`, `sugerencia` y `otro`.

<Info>
  Al igual que `POST /api/submit`, la IP del cliente se captura desde el header `CF-Connecting-IP` de Cloudflare y se almacena en la tabla `soporte` de D1.
</Info>

## POST /api/soporte

### Request

**Content-Type:** `multipart/form-data`

<ParamField body="nombre" type="string" required>
  Nombre completo de la persona que envía el ticket.
</ParamField>

<ParamField body="correo" type="string" required>
  Dirección de correo electrónico de contacto. Debe ser una dirección de email válida.
</ParamField>

<ParamField body="tipo" type="string">
  Categoría del ticket. Valores aceptados:

  - `consulta` — Pregunta o duda general
  - `error` — Reporte de fallo o problema técnico
  - `sugerencia` — Propuesta de mejora
  - `otro` — Cualquier otro tipo de solicitud
</ParamField>

<ParamField body="mensaje" type="string" required>
  Descripción detallada del problema, consulta o sugerencia.
</ParamField>

### Responses

<ResponseField name="ok" type="boolean" required>
  `true` si el ticket fue guardado correctamente, `false` en caso de error.
</ResponseField>

<ResponseField name="message" type="string">
  Presente en respuestas exitosas. Valor: `"Mensaje enviado"`.
</ResponseField>

<ResponseField name="error" type="string">
  Presente en respuestas de error. Describe el motivo del fallo.
</ResponseField>

| Código | Descripción |
|---|---|
| `200` | Ticket de soporte enviado correctamente |
| `400` | Faltan campos obligatorios (`nombre`, `correo` o `mensaje`) |
| `500` | Error interno del servidor |

### Ejemplo

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://servitecpma.com/api/soporte \
  --form "nombre=Ana Torres" \
  --form "correo=ana@ejemplo.com" \
  --form "tipo=error" \
  --form "mensaje=La página de inicio no carga correctamente en Safari"
```

</CodeGroup>

<ResponseExample>

```json 200
{
  "ok": true,
  "message": "Mensaje enviado"
}
```

```json 400
{
  "ok": false,
  "error": "Campos obligatorios faltantes"
}
```

```json 500
{
  "ok": false,
  "error": "Error interno"
}
```

</ResponseExample>

## GET /api/admin-soporte

Retorna todos los tickets de soporte almacenados. Este endpoint es consumido por la vista **Ver Soporte** del panel de administración.

<Warning>
  Al igual que `GET /api/admin`, este endpoint **no tiene autenticación server-side**. Cualquier cliente puede acceder a los datos directamente sin presentar un token.
</Warning>

### Request

No requiere parámetros ni headers de autenticación.

### Response

Retorna un arreglo de objetos ticket con los siguientes campos:

<ResponseField name="id" type="number" required>
  Identificador único autoincremental del ticket en la base de datos.
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Nombre de la persona que envió el ticket.
</ResponseField>

<ResponseField name="correo" type="string" required>
  Correo electrónico de contacto del remitente.
</ResponseField>

<ResponseField name="tipo" type="string" required>
  Categoría del ticket: `consulta`, `error`, `sugerencia` o `otro`.
</ResponseField>

<ResponseField name="mensaje" type="string" required>
  Contenido del ticket enviado.
</ResponseField>

<ResponseField name="fecha" type="string" required>
  Timestamp de cuando se registró el ticket. Formato ISO 8601.
</ResponseField>

### Códigos de respuesta

| Código | Descripción |
|---|---|
| `200` | Arreglo de tickets retornado correctamente |
| `500` | Error al consultar la base de datos |

### Ejemplo

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://servitecpma.com/api/admin-soporte
```

</CodeGroup>

<ResponseExample>

